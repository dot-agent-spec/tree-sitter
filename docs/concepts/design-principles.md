# Design Principles

The .agent DSL is designed for human readability and AI generability, prioritizing clarity over syntactic power.

## 1. Core Constraints

Both `.description` and `.behavior` adhere to three foundational rules:

1. **Zero Noise**: No curly braces `{}`, mandatory commas, or colons in key-value pairs. The DSL reads like a textual diagram.
2. **Semantic Indentation**: Scope is defined by 2-space indentation. This reduces visual clutter and simplifies the parsing model for LLMs.
3. **Strict Determinism**: Data structures must be explicitly anchored to semantic URIs (Wikidata/Schema.org). The Runtime never "guesses" a type's schema.

## 2. .behavior Design Principles

The reserved word count is a strategic decision, not just a syntactic choice. A smaller vocabulary ensures the language is faster to learn, more predictable when generated by AI, and easier to document.

Every proposed addition must answer: *"Does this reduce cognitive load, or does it only add power?"* Power alone is not justification; complexity belongs in `.wasm`.

**What .behavior is not:**
- **Not a programming language**: No loops, recursion, or arithmetic. Complexity belongs in `.wasm`.
- **Not YAML/JSON**: It is a behavioral language, not a data serialization format.
- **Not Natural Language**: It uses human concepts with structured syntax to ensure high-fidelity parsing and prevent "hallucinated" formatting.

## 3. Design Inspirations

| Origin | Contribution to .agent |
|---|---|
| **Statecharts** | Parallel states and global triggers (`on event`). |
| **Behavior Trees** | Goal-oriented logic and failure resilience (`on failed`). |
| **AWS Step Functions** | Strict input/output isolation for state nodes. |
| **GitHub Actions** | Readable, top-down sequential block structure. |

**Why not YAML?**
Strict YAML was abandoned because its syntactic noise (hyphens, mandatory quotes, deep nesting) encourages formatting errors in smaller LLMs and reduces scannability for non-developers.
