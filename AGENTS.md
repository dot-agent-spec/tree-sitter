## What this package is

`tree-sitter-agent` provides the **canonical grammar source** for the two file formats of the Agent DSL:

| Grammar | Covers | Located in |
|---------|--------|------------|
| Root `grammar.js` | `.agent` manifests and `.type` declarations | `grammar.js`, `src/` |
| `flow/grammar.js` | `.flow` behavior files | `flow/grammar.js`, `flow/src/` |

Both grammars are authored here and compiled by Tree-sitter CLI into C parsers that power IDE tooling (syntax highlighting, LSP diagnostics, go-to-definition) across VS Code, Zed, Neovim, and Helix. The language specification lives in [`language.md`](https://github.com/daniloborges/dot-agent/blob/main/dsl/language.md); this package is the executable implementation of that spec.

## Development workflow

```
1. Edit grammar.js (or flow/grammar.js)
       â†“
2. npx tree-sitter generate          # recompiles src/parser.c
       â†“
3. npx tree-sitter parse <file> --quiet   # manual smoke test
       â†“
4. Update test/corpus/ if needed
       â†“
5. npx tree-sitter test              # confirm all corpus cases pass
```

For the flow grammar, run the same steps inside `flow/` using `npx tree-sitter generate` and `npx tree-sitter test` from that directory (or use the npm scripts: `npm run generate-flow`, `npm run test-flow`).

**After editing `grammar.js`**, always regenerate before committing â€” stale `src/parser.c` causes silent parse failures in editors.

## Grammar architecture notes

Both grammars share the same indentation strategy: INDENT, DEDENT, and NEWLINE tokens are emitted by an **external scanner** (`src/scanner.c` / `flow/src/scanner.c`) because Tree-sitter has no native indent support. The scanner uses a stack to track indent levels and emits the correct synthetic token per line boundary.

The sep-by pattern used in blocks:
```js
seq(item, repeat(seq($._newline, item)), $._dedent)
```
is intentional â€” items must **not** consume their own trailing newline because the scanner emits `_dedent` (not `_newline`) after the last item in an indented block.

## Adding new syntax

1. Add the rule to `grammar.js` (or `flow/grammar.js`)
2. Add it to the appropriate parent rule's `choice(...)` list
3. Regenerate: `npx tree-sitter generate`
4. Add a corpus test case in `test/corpus/` illustrating the new syntax
5. Update `queries/highlights.scm` with appropriate capture names
6. If this represents a spec change, update [`language.md`](https://github.com/daniloborges/dot-agent/blob/main/dsl/language.md) in the `dot-agent` repo
